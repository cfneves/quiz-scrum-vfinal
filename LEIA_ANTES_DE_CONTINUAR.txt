━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. EMBARALHAMENTO DETERMINÍSTICO
   random.Random(username).shuffle(perguntas)
   Cada aluno vê sempre a mesma ordem. Ordensdiferentes entre alunos.
   As perguntas embaralhadas ficam em session_state.perguntas_embaralhadas
   para que results_ui use a mesma ordem que quiz_ui.

2. SESSION STATE
   Todas as chaves são inicializadas em SESSION_DEFAULTS (config.py).
   app.py inicializa no boot:
     for _k, _v in SESSION_DEFAULTS.items():
         if _k not in st.session_state: st.session_state[_k] = _v
   Chaves: autenticado, usuario, role, cadastrando, index,
           respostas, quiz_finalizado, perguntas_embaralhadas

3. LAYOUT
   Aluno: colunas [1,2,1] — coluna central de 50% da largura
   Professor: layout="wide" sem restrição de coluna (usa tela toda)
   CSS premium em components/styles.py injetado via st.markdown unsafe

4. MOCK DE TESTES
   conftest.py cria _st_mock que substitui o módulo streamlit nos testes.
   Suporta @st.cache_data (sem args) e @st.cache_data(ttl=60) (com args).
   Isso permite testar quiz_service e auth_service sem Streamlit rodando.

5. PROFESSOR: DUPLA VERIFICAÇÃO
   a) autenticar(username, senha) → verifica senha
   b) username in PROFESSOR_USUARIOS → verifica autorização
   Ambos precisam ser True para acessar o painel.

6. SHEETS: FALHA SILENCIOSA
   sheets_service.enviar_respostas() e ler_resultados() nunca levantam
   exceção para o usuário — apenas retornam False ou DataFrame vazio.
   O CSV local é o primário; Sheets é bônus/backup.

7. REQUIREMENTS.TXT
   Estava corrompido em UTF-16 (espaço entre cada caractere).
   Foi reescrito em UTF-8 correto. Esse foi o bug que causava
   "Error installing requirements" no Streamlit Cloud.
